# ITN 处理器

## 任务概述
对选中的CSV进行ITN转写。

## 核心流程

### 1. 读取 CSV 文件
- 获取文件中的方言文本（xx话, 一般为列第一列）
- 编码格式：UTF-8 with BOM

### 2. 识别可能需要 ITN 转换的行
1. 增加原行号标记
2. **Python 自动扫描 `xx话` 列，**
   - 检测中文数字：零、一、二、三、四、五、六、七、八、九、十、百、千、万、亿、两
   - 检测数量词：米、公里、分钟、小时、秒、度、元、块、角、分等
   - 检测序号词：第、初、几、号等
   - 检测时间词：点、分、秒、刻、半等
3. **添加 `need_itn` 标记列**（布尔值）：
   - `True` = 包含数字相关词汇，可能需要进行 ITN 转写
   - `False` = 无数字相关词汇，无需转写
4. 筛选`need_itn`为`False`的列, 直接复制方言文本到itn_reasult列, 落盘为中间文件1 `output/{原文件名}_no_itn.csv`
5. 筛选`need_itn`为`True`的列, `itn_reasult`列留空, 落盘为中间文件2 `output/{原文件名}_itned.csv`


### 3. 分批次进行 ITN 转写

**分批策略**: 每次处理 20 行数据，**只处理已标记的行**

**处理步骤**:
1. 读取中间文件2`output/{原文件名}_itned.csv`中的所有记录
2. 筛选其中`itn_reasult`列为空的行
3. **按顺序分批处理**，每批 20 行
4. 对每批数据中的每条记录：
   - LLM 判断哪些数字需要转换，哪些需要保留
   - LLM 执行 ITN 转写，生成 `itn_reasult` 值
5. 逐条检查质量，确保符合规范
6. 完成一批后存储csv, 继续下一批，直到所有数据处理完毕

**异常处理策略**：
- 某一行转写失败：标记该行，继续处理同批次其他行
- 单批次错误率超过 50%：停止处理，检查共性问题
- 连续 3 批出现相同错误：暂停并报告用户

### 4. 拼接 `output/{原文件名}_itned.csv`和`output/{原文件名}_no_itn.csv`, 按照原行号排序, 生成 `output/{原文件名}_itn_processed.csv`（保留所有列）, 删除过程文件

### 5. ITN 转写规则

对方言文本列内容进行转换，遵循以下规则：

#### 通用原则

- **符合书面语和视觉习惯**为基本准则
- **并列、起止范围保持前后一致**：如"我从三四点一直走到7点"→"我从三四点一直走到七点"

#### 必须转换的类型

**数字**：
- 整数：十一→11，一百零一→101，一万二千→12000
- 大数单位：十万→10万，四百万→400万，十四亿→14亿
- 小数：一点七五→1.75，三十二点八→32.8，一百零八点二→108.2
- 分数：三分之二→2/3，五分之四→4/5，3分之2→2/3
- 百分数：百分之二十→20%，百分之三十四点五八→34.58%
- 千分数：千分之二→2‰，千分之二点五→2.5‰
- 比例：一比三→1:3，十比一→10:1，一比二比三→1:2:3
- 数字范围：三到六个→3~6个，五到十米→5~10米（连接词两端为数字时）

**日期时间**：
- 年份：二零二四年→2024年，零八年→08年，两千零二年→2002年
- 日期：五月一日→5月1日，十月二十号→10月20号
- 世纪年代：二十一世纪→21世纪，九十年代→90年代，二十世纪八十年代→20世纪80年代
- 时间：下午三点二十→下午3点20，两点半→2点半，九点一刻→9点一刻

**度量衡**：
- 长度：五千米→5千米，一米八→1米8，五十厘米→50厘米
- 温度：一百摄氏度→100摄氏度，三十八度五→38度5
- 重量：五百克→500克，一千克→1千克
- 速度：每小时八十公里→每小时80公里
- 时长：三个小时→3个小时
- 容量：二十点五升→20.5升，五百毫升→500毫升
- 数据：二十兆流量→20兆流量
- 物理：一百千帕→100千帕，两百赫兹→200赫兹，三百焦耳→300焦耳

**货币**：
- 八块钱→8块钱，一百五十元→150元
- 两万块→2万块，三十六万六千元→36万6000元，三亿五千万元→3亿5000万元
- 五块六毛→5块6毛，三块五→3块5
- 外币：十二点五欧元→12.5欧元，三千英镑→3000英镑，一百日元→100日元

**量词规则**：
- **确数（精确计数）**：一二三及以上量词均可转数字
  - 三辆车→3辆车，五本书→5本书，卖了3辆汽车
  - 一碗饭→1碗饭，两杯水→2杯水（精确计数时可转）
- **虚指/泛指（保留汉字）**：一、二后接量词保留
  - 说两句、有一点湿、有一天下午（非精确计数）
- 序数词：第三层→第3层，第五题→第5题（但"第一中学"保留）

**通信号码（逐位转换）**：
- 手机：幺五九幺零零→159100（"幺"转1）
- 固话：零一零杠八八八→010-888（"杠"转"-"）
- IP：一九二点一六八→192.168（"点"转"."）
- QQ/验证码：六五四三二一→654321，验证码四零二九→验证码4029
- 国际区号：加八六→+86（首位"加"转+）
- 分机：保留"转"字，如010-888转606

**地点编号**：
- 门牌：甲十一号→甲11号，十八层→18层，三零二室→302室
- 楼栋单元：五号楼二单元→5号楼2单元
- 楼层：地下/负一层→地下/负1层（保留前缀词）
- 交通线：二号线→2号线，幺四九路→149路，二零一国道→201国道
- 座位：六A座位→6A座位，三车厢五十八号座→3车厢58号座
- 邮编：六五零零一一→650011

**特殊编码（逐位转换）**：
- 身份证：零八五八二叉→08582X（"叉"转X）
- 车牌：粤B五九幺六幺→粤B59161
- 航班：MU五七零二→MU5702
- 火车：G一百零一→G101，K八十八→K88，高铁G一百零一→高铁G101
- 地点字母编号：商场D二区→商场D2区

**产品型号和标准化名词**：
- 产品型号：iPhone十六→iPhone 16，小米十三→小米13，小米SU7，特斯拉Model 3
- 参数：5座SUV，8缸发动机，6速变速箱，14英寸MacBook Pro，12GB内存
- 标准化名词：4S店，911事件，5G通讯，4K电视，A4纸，985大学，C9联盟，东风41，5A景区，标普500指数，3D打印，N95口罩，波音737，歼-20

**特殊格式**：
- 字母数字混合：G310国道→G310国道（保持原样）
- 罗马数字：III→III（保留原样，除非明确表示"三"）
- 化学分子式：H2O→H2O（保持H2O或H₂O）
- 数学表达式：x平方→x²或x^2

#### 禁止转换的情况（保留原样）

**专有名词**：
- 地名景点：九寨沟、三里屯、明十三陵、龙门石窟、万里长城
- 作品名：三国演义、七里香、五月天、一千零一夜、海底两万里
- 品牌：三只松鼠、天下第一关
- 其他专有名词：英语四级、科目三考试、二次元、三位一体、三维空间、六度空间理论、第一学历、四大天王、十三朝古都

**成语固定搭配**：
- 十之八九、一五一十、十年磨一剑、午时三刻
- 一年一度、一传十十传百、十万个为什么、十年寒窗、七年之痒、四书五经、一年三熟
- 八百里分麾下炙、五十弦翻塞外声

**星期和农历**：
- 星期：星期六、周一、礼拜五
- 农历节气：正月十五、腊月初八、正月初八
- 历史年号：洪武十三年、万历十五年

**节日事件**：
- 十月革命、七七事变、十一黄金周、五一劳动节

**量词规则（虚指时保留）**：
- 一、二后接量词在虚指/泛指时保留：说两句、有一点湿、有一天下午
- 特定量词组：半盖、半片、半拉、半边天、半天、半辈子、半路、半推半就

**虚指泛指**：
- 一天下午、一个清晨、有一年夏天、他一个人生活、我们是两个世界的人
- 有一点湿、说两句、玩两把、一气之下、玩两天、过两天再说
- 一男子、两个闲钱、晚一点、一到路上

**概数模糊**：
- 相邻数字概数：三四百、两三千、七八十万块、十五六斤、千八百块、五六岁、四五点钟、初七八、三五天
- "几"组成的概数：十几个、几十个、几百个、三十几个（"30几个"视觉不协调）
- "十"和"多/几/来/余"组成：十多个、十来个、十余个
- 等两天、过两天

**叙事与修饰性描述**：
- 你的眼睛像天边的两颗星
- 精神上百分之百支持、百分之一万没问题、放一百个心

**数学运算（不转公式）**：
- 二又三分之一→2又3分之1（不转2+1/3）
- 1加1等于2、2的3次方
- x加5除以2等于10（不能转x+5/2=10）

### 6. 格式要求（极其重要 - 强制执行）

**核心原则：数字/字母和汉字之间绝对不加空格**

这是 ITN 转写的**基本要求**，必须严格遵守！

#### 快速处理策略（推荐）

**如果文本中只有中文（无英文、无数字字母混合）**：
- ✅ **直接删除所有空格** - 最简单可靠的方案
- 示例：`"直行 100 米"` → 删除空格 → `"直行100米"` ✅
- 示例：`"G 101 国道"` → 不适用此策略（含字母）

**判断标准**：
- 文本中只包含汉字、数字、中文标点 → 可用快速策略
- 文本中包含英文字母、英文标点 → 需按详细规则处理

#### 正确示例（无空格）
- ✅ `100米` - 数字和单位之间无空格
- ✅ `300米` - 数字和量词之间无空格  
- ✅ `G101` - 字母和数字之间无空格
- ✅ `MU5702` - 航班号字母数字紧密连接
- ✅ `粤B59161` - 车牌号汉字、字母、数字之间无空格
- ✅ `2号线` - 数字和名词之间无空格
- ✅ `30秒` - 数字和时间单位无空格
- ✅ `8块钱` - 数字和货币单位无空格
- ✅ `5公里` - 数字和距离单位无空格

#### 错误示例（严禁出现）
- ❌ `100 米` - 数字和单位之间有空格
- ❌ `300 米` - 数字和量词之间有空格
- ❌ `G 101` - 字母和数字之间有空格
- ❌ `MU 5702` - 航班号中间有空格
- ❌ `粤 B 59161` - 车牌号各部分之间有空格
- ❌ `2 号线` - 数字和名词之间有空格
- ❌ `30 秒` - 数字和时间单位之间有空格

#### 转换规则详解

**规则 1：数字 + 单位 = 无空格**
```
输入：直行一百米
输出：直行100米  ✅
错误：直行 100 米  ❌
```

**规则 2：字母 + 数字 = 无空格**
```
输入：G 一百零一
输出：G101  ✅
错误：G 101  ❌
```

**规则 3：汉字 + 字母 + 数字混合 = 全部无空格**
```
输入：粤 B 五九幺六幺
输出：粤B59161  ✅
错误：粤 B 59161  ❌
错误：粤 B59161   ❌
```

**规则 4：特殊符号也要无空格**
```
输入：零一零杠八八八
输出：010-888  ✅ （连字符前后无空格）
输入：一九二点一六八  
输出：192.168  ✅ （小数点前后无空格）
```

#### 质量检查要点

每完成一批 ITN 转写后，必须检查：
1. ✅ 所有数字和汉字之间是否有空格？
2. ✅ 所有字母和数字之间是否有空格？
3. ✅ 所有特殊符号（如连字符、小数点）前后是否有空格？
4. ✅ 使用文本编辑器的查找功能搜索空格，确认没有误加

#### 空格清理脚本（Python）

```python
def clean_spaces(text):
    """
    清理 ITN 转写结果中的空格
    策略：如果文本中只有中文和数字，直接删除所有空格
    """
    if not text or pd.isna(text):
        return text
    
    text = str(text)
    
    # 检查是否包含英文字母或英文标点
    has_english = any(c.isalpha() and ord(c) < 128 for c in text)
    has_english_punct = any(c in '.,;:!?()[]{}@#$%^&*' for c in text)
    
    if not has_english and not has_english_punct:
        # 纯中文环境：直接删除所有空格
        return text.replace(' ', '').replace('\u3000', '')
    else:
        # 含英文/字母：按详细规则处理
        # 保留字母数字间的空格（如 "G 101" 需要变成 "G101"）
        import re
        # 删除数字与汉字间的空格
        text = re.sub(r'(\d)\s+([\u4e00-\u9fff])', r'\1\2', text)
        text = re.sub(r'([\u4e00-\u9fff])\s+(\d)', r'\1\2', text)
        # 删除字母与数字间的空格
        text = re.sub(r'([a-zA-Z])\s+(\d)', r'\1\2', text)
        text = re.sub(r'(\d)\s+([a-zA-Z])', r'\1\2', text)
        return text
```

**违规后果**：
- 格式不符合标准
- 可能被下游系统拒绝
- 影响数据质量和专业性

### 7. 处理方式要求（强制）

**必须使用 LLM 进行 ITN 转写，严禁使用 Python 脚本或其他自动化工具**：

- ✅ **必须**：由 LLM 逐条理解语境并进行 ITN 转写
- ❌ **禁止**：使用 Python 脚本、正则表达式替换等自动化方式
- ❌ **禁止**：使用外部 ITN 工具包（如 WeTextProcessing 等）

**原因**：
1. **语境理解** - LLM 能准确判断歧义（如"三点零五"是时间还是数值）
2. **语义保护** - LLM 能识别专有名词、成语、概数等需要保留的情况
3. **方言处理** - LLM 能理解并保留方言词汇（嫑、甭、恁等）
4. **质量保证** - LLM 处理准确率远高于规则引擎

**LLM 处理流程**：
```
步骤 1 - 生成中间文件:
1. 读取 data/ 目录下所有 CSV 文件
2. 从文件名提取城市代码
3. 为每条记录生成 UID（跳过 transcription 为空的行）
4. 创建中间文件 output/{原文件名}（含 UID，无 ITN）
5. 验证：检查是否有空行，如有则删除并重新编号

步骤 2 - 分批 ITN 转写:
1. 读取中间文件中的所有记录
2. 按顺序分批处理，每批 20 行
3. 对每批中的每条记录:
   - LLM 理解 transcription 列的完整语境
   - LLM 判断哪些数字需要转换，哪些需要保留
   - LLM 执行 ITN 转写，生成 transcription_itn 值
   - **立即验证**: 检查 transcription 和 ITN 是否匹配（长度差异≤5 字符）
4. 完成一批后：
   - 逐条检查质量
   - 随机抽样 3-5 行复核
   - 确认无误后继续下一批
5. 所有批次完成后，生成最终文件

步骤 3 - 最终清理与验证:
1. 删除所有 transcription 为空的行
2. 重新生成连续的 index 和 UID
3. 执行阶段三的所有质量检查项
4. 生成最终交付文件
```

## 质量检查清单

### 阶段一检查（生成中间文件后）
- [ ] 中间文件已创建在 `output/` 目录
- [ ] 中间文件包含 6 列：`index`, `uid`, `transcription`, `translation`, `gloss`, `need_itn`
- [ ] UID 格式正确（uid_+6 位城市代码 +001+8 位序号）
- [ ] UID 连续不重复
- [ ] 城市代码准确（优先查表，无表则合理推断）
- [ ] 输出文件 UTF-8 with BOM 编码
  **编码验证方法**：
  - Python 检测：`import codecs; codecs.open(file, 'r', 'utf-8-sig')`
  - Excel 打开能正常显示即为 UTF-8 with BOM
  - 或使用文本编辑器（VSCode、Notepad++）查看编码格式
- [ ] **过滤空行** - 删除 `transcription` 列为空的记录
- [ ] **need_itn 标记准确** - 包含数字相关词汇的行已标记为 True

### 阶段二检查（分批 ITN 处理后）
- [ ] 所有批次已处理完成（每批 20 行）
- [ ] 专有名词未转换（路名、景点、作品名）
- [ ] 精确数字已转换（计数、金额、时间）
- [ ] 通信号码逐位转换（手机号、固话、IP）
- [ ] 特殊代称处理（"幺"→1，"叉"→X）
- [ ] 概数虚指保留原样
- [ ] 方言词汇保留（嫑、甭、恁等）
- [ ] **数字/字母和汉字之间无空格**（🔴 核心检查项 - 必须逐项检查）
  - [ ] 检查所有数字与汉字之间是否有空格
  - [ ] 检查所有字母与数字之间是否有空格
  - [ ] 检查所有特殊符号（- . /）前后是否有空格
  - [ ] 使用查找功能搜索空格字符确认
- [ ] **ITN 转写由 LLM 完成**（强制检查项）
- [ ] 最终文件包含完整的 6 列
- [ ] CSV 列顺序正确
- [ ] **长度差异检查** - transcription 和 ITN 的长度差异应在±5 字符范围内（绝对值≤5）
- [ ] **逐行验证** - transcription 和 transcription_itn 必须对应同一句话

### LLM 逐条处理规范（确保结果正确）

**处理每一条时，必须**：

1. **理解完整语境** - 阅读整句话，判断数字是确数还是虚指
2. **参照规范判断** - 对照 ITN 标注规范决定是否转换
3. **直接输出结果** - 不要输出思考过程，只输出转换后的文本
4. **保持原格式** - 保留原文中的换行符、标点符号

**确数 vs 虚指判断要点**：

| 类型 | 特征 | 示例 | 处理 |
|------|------|------|------|
| **确数** | 精确计数、时间、金额、度量 | "3辆车"、"5块钱"、"10年" | 转阿拉伯数字 |
| **虚指** | 泛指、模糊、习惯性表达 | "说两句"、"有一天"、"过两天" | 保留汉字 |
| **概数** | 约数、不确定 | "三四百"、"百把米"、"十几个" | 保留汉字 |

**常见陷阱**：
- ❌ "一个年轻人"（虚指，保留） vs ✅ "1个钟头"（确数，转换）
- ❌ "歇两日"（概数，保留） vs ✅ "2块钱"（确数，转换）
- ❌ "一眼"（差一点，保留） vs ✅ "1只包"（确数，转换）

**输出格式要求**：
```
输入: 我有十年工夫啊，靠老婆三十块八角过日脚啊。
输出: 我有10年工夫啊，靠老婆30块8角过日脚啊。

输入: 侬叫一个年轻人，叫伊讲一个钟头
输出: 侬叫一个年轻人，叫伊讲1个钟头
```

import pandas as pd
import re

def check_spaces(file_path, text_col='itn_result'):
    """
    检查 ITN 结果中的空格问题
    返回包含空格的行列表
    """
    df = pd.read_csv(file_path, encoding='utf-8-sig')
    issues = []
    
    for idx, row in df.iterrows():
        text = str(row.get(text_col, ''))
        
        # 检查数字与汉字间空格
        if re.search(r'\d\s+[\u4e00-\u9fff]|[\u4e00-\u9fff]\s+\d', text):
            issues.append({
                'row': idx + 2,
                'type': '数字-汉字间空格',
                'content': text
            })

            issues.append({
                'row': idx + 2,
                'type': '字母-数字间空格',
                'content': text
            })
    
    return issues

### 脚本 2：内容匹配检查

```python
def check_alignment(file_path, orig_col='方言文本', itn_col='itn_result'):
    """
    检查 transcription 和 ITN 是否匹配
    通过长度差异和关键词匹配判断
    """
    df = pd.read_csv(file_path, encoding='utf-8-sig')
    mismatches = []
    
    for idx, row in df.iterrows():
        orig = str(row.get(orig_col, ''))
        itn = str(row.get(itn_col, ''))
        
        # 长度差异检查
        len_diff = abs(len(orig) - len(itn))
        if len_diff > 10:  # 差异过大
            mismatches.append({
                'row': idx + 2,
                'reason': f'长度差异过大({len_diff})',
                'original': orig,
                'itn': itn
            })
            continue
        
        # 提取关键词（非数字汉字）检查
        orig_keywords = set(re.findall(r'[\u4e00-\u9fff]{2,}', orig))
        itn_keywords = set(re.findall(r'[\u4e00-\u9fff]{2,}', itn))
        
        # 检查关键方言词汇是否保留
        lost_keywords = orig_keywords - itn_keywords
        if len(lost_keywords) > 2:  # 丢失太多关键词
            mismatches.append({
                'row': idx + 2,
                'reason': f'关键词丢失: {lost_keywords}',
                'original': orig,
                'itn': itn
            })
    
    return mismatches
```

### 脚本 3：快速修复空格

```python
def fix_spaces_in_csv(input_path, output_path, text_col='itn_result'):
    """
    快速修复 CSV 中的所有空格问题
    策略：纯中文直接删空格，含英文按规则处理
    """
    df = pd.read_csv(input_path, encoding='utf-8-sig')
    
    def clean_cell(text):
        if pd.isna(text):
            return text
        text = str(text)
        
        # 检查是否包含英文字母
        has_english = any(c.isalpha() and ord(c) < 128 for c in text)
        
        if not has_english:
            # 纯中文：直接删除所有空格
            return text.replace(' ', '').replace('\u3000', '')
        else:
            # 含英文：精细处理
            text = re.sub(r'(\d)\s+([\u4e00-\u9fff])', r'\1\2', text)
            text = re.sub(r'([\u4e00-\u9fff])\s+(\d)', r'\1\2', text)
            text = re.sub(r'([a-zA-Z])\s+(\d)', r'\1\2', text)
            text = re.sub(r'(\d)\s+([a-zA-Z])', r'\1\2', text)
            return text
    
    df[text_col] = df[text_col].apply(clean_cell)
    df.to_csv(output_path, index=False, encoding='utf-8-sig')
    print(f"✅ 已修复并保存到: {output_path}")

# 使用示例
fix_spaces_in_csv('output/result_itn_processed.csv', 'output/result_fixed.csv')
```

4. **最终清理** - 所有修正完成后执行阶段三的完整验证
